I have been working on a couple of [Omeka](http://omeka.org/) sites this semester. I really like the Omeka system for its handling of meta data and the [Neatline](http://neatline.org/) map extension for easy map creation. Omeka S just came out and looks fantastic, though I haven’t had a chance to build with it yet.

[!\[\](http://www.johnastewart.org/wp-content/uploads/2017/12/Screen-Shot-2017-12-04-at-4.33.28-PM-300x222.png)](http://www.johnastewart.org/wp-content/uploads/2017/12/Screen-Shot-2017-12-04-at-4.33.28-PM.png) However, I do not like the Omeka themes. For a while I was using [Denison](https://github.com/Orpheusp/denison-expanding-archives-omeka-theme) as seen here. However, the theme mishandles the four images on the front page and the drop down menu has a tendency to malfunction.

I also tried [Big Picture](https://omeka.org/classic/themes/bigpicture/) but abandoned it when I was unable to merge the beautiful Browse Collections page into the index page.

Ultimately I turned away from Omeka themes and instead used [HTML5Up](https://html5up.net/) templates. I transitioned [Dr. Daniela Busiciglio’s](http://modlang.ou.edu/daniela-busciglio) site that had been built with Denison from this:

[!\[\](http://www.johnastewart.org/wp-content/uploads/2017/09/Screen-Shot-2017-09-13-at-4.42.26-PM-1024x702.png)](http://www.johnastewart.org/wp-content/uploads/2017/09/Screen-Shot-2017-09-13-at-4.42.26-PM.png)

to this ( [progettovitaliano.org](http://progettovitaliano.com/) ):

[!\[Screen Shot 2017-12-04 of progettovitaliano.com\](http://www.johnastewart.org/wp-content/uploads/2017/12/Screen-Shot-2017-12-04-at-4.38.30-PM-1024x758.png)](http://www.johnastewart.org/wp-content/uploads/2017/12/Screen-Shot-2017-12-04-at-4.38.30-PM.png)

I’ve just finished transitioning another site that I built with Dr. Mirelsie Velazquez and her students from the Big Picture Omeka theme to this ( [http://docnarr.oucreate.com/](http://docnarr.oucreate.com/) ):

[!\[\](http://www.johnastewart.org/wp-content/uploads/2017/12/Screen-Shot-2017-12-04-at-4.43.00-PM-1024x758.png)](http://www.johnastewart.org/wp-content/uploads/2017/12/Screen-Shot-2017-12-04-at-4.43.00-PM.png)

Because Omeka’s API does not provide images, I was not able to load the images into these HTML based pages programmatically. Thus, these front pages will require a bit more maintenance in their HTML code than Omeka themes, which are controlled from the administrative GUI. However, I really like the flexibility that I get from stripping out the front page of the Omeka themes and replacing it with something that I can tweak to feature the best parts of the projects.
